# Дата: 2026-03-29

## Что было сделано

Сегодня добила новый виджет — Code Completion.

Идея простая: даём пользователю небольшой кусок JS-кода с пропусками, и он должен их заполнить. Такой мини-тренажёр на подумать, без перегруза, но с пользой.

Сам компонент разложила нормально по полочкам:
— вынесла типы отдельно (levels, solution и т.д.)
— логику в кастомный хук (`useCodeCompletionLogic`)
— UI разбила на компоненты: редактор, подсказки, модалка

В итоге сам `CodeCompletionGame` выглядит довольно чисто и читабельно — там уже больше про сборку всего вместе, чем про логику.

Что внутри:
— инпуты для пропусков прямо в коде (через CodeEditor)
— подсказки в HintPanel
— таймер (переиспользовала из AsyncSorter — приятно, когда старый код живёт дальше)
— проверка ответа через `onValidate`
— состояния win / lose с модалкой и роботами

Добавила немного геймификации:
если успел и правильно — победа + передаём время наверх
если нет — показываем луз и даём попробовать ещё раз

Ещё важный момент — всё завязано на i18n, так что тексты сразу через `t()`, без хардкода.

Отдельно пришлось поправить роутинг, чтобы новый виджет нормально встраивался в приложение и можно было переключаться между уровнями. Там чуть подправила структуру, чтобы всё было консистентно с остальными виджетами.

В целом получилось аккуратно:
логика отделена, компоненты переиспользуемые, структура масштабируемая — можно спокойно наращивать уровни и усложнять задания дальше.
