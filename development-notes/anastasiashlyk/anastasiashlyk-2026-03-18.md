# Дата: 2026-03-18

Сегодня я занялась реализацией ввода сообщений голосом. То есть вместо того, чтобы писать сообщение, можно его надиктовать. Для этого буду использовать Web Speech API.
Для начала нужно хотя бы базово разобраться в его работе. Web Speech API имеет два основных интерфейса:
SpeechRecognition — превращает аудио в текст
SpeechSynthesis — превращает текст в аудио
Хочу попробовать озвучивать ответы ИИ-агента.
Также сегодня занялась сохранением состояния чата при перезагрузке страницы. Для этого нужно сохранять conversationId в localStorage и при каждой перезагрузке загружать историю чата из базы данных. Для этого я добавила эндпоинт в AiController, чтобы можно было получать сообщения по conversationId.

## Что было сделано

- Добавлен Speech Recognition, теперь можно надиктовать сообщение.
- Ответ от ИИ-агента теперь озвучивается с помощью speechSynthesis.
- Исправлен интерфейс после последнего мержа — чат снова выглядит корректно.
- Реализована загрузка истории сообщений по conversationId из localStorage, теперь при перезагрузке страницы сообщения не пропадают.
- Настроен backend, чтобы получать историю сообщений и создавать новые разговоры, если conversationId отсутствует.
- Реализована интеграция с frontend:
  - conversationId сохраняется в localStorage
  - при загрузке страницы проверяется наличие существующего conversationId
  - если он есть, загружается история сообщений в хронологическом порядке
  - добавлен кастомный хук useChat с useChatMessages, который управляет состоянием чата
- исправлены предупреждения ESLint

## Проблемы

- Агент не перестает озвучивать сообщение, пока оно полностью не проговорено. При нажатии на кнопку диктовки хочется, чтобы текущая озвучка останавливалась, если пользователь запускает новую запись. Пока не пойму как это сделать.
