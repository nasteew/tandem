## Описание моей работы

В рамках проекта я выступала в роли тимлида и занималась разработкой как backend, так и frontend частей приложения. Основной моей задачей было создание безопасной системы авторизации, разработка страницы профиля пользователя, а также реализация widget engine и интерактивного обучающего виджета для тренировки навыков. Кроме того, я занималась разработкой и интеграцией ключевых backend-сервисов для страницы профиля, глобальной статистики, статистики по виджетам и логики работы widget engine.

## Основной вклад

### 1. Система авторизации
Я разработала полноценный сервис авторизации на backend с использованием NestJS. Реализовала:
- регистрацию и логин пользователей
- работу с JWT (access и refresh токены)
- безопасное обновление токенов
- хранение refresh-токена в cookie
- logout (инвалидация токена)
- защиту приватных маршрутов
- ограничение запросов (Throttler) для защиты от атак

Также я реализовала авторизацию через Google. Это включало:
- работу с OAuth
- реализацию авторизации через popup-окно вместо редиректа
- безопасную передачу данных между окнами
- обработку кейсов (например, если пользователь зарегистрирован через Google, но пытается войти через email)

### 2. Backend (Users + API)
Я разработала и доработала backend-часть для работы с пользователями:
- получение профиля текущего пользователя
- обновление данных (имя, email, пароль)
- удаление аккаунта
- рефакторинг структуры (объединение обновлений в один endpoint)
- добавление документации через Swagger


### 3. Страница профиля (Frontend)
Я реализовала страницу профиля на React:
- спроектировала UI/UX страницы
- реализовала просмотр и редактирование данных
- добавила смену пароля / установку пароля для Google-пользователей
- реализовала удаление аккаунта
- подключила backend

Для управления состоянием использовала Zustand:
- хранение userId
- хранение ссылки на аватар
- глобальный доступ к данным

### 4. Загрузка аватара (Supabase)
Я реализовала загрузку и хранение аватаров пользователей:
- выбрала платформу (Supabase) после сравнения нескольких решений
- настроила загрузку файлов в Storage
- реализовала получение публичной ссылки
- сохранила ссылку в базе данных
- связала backend и frontend

### 5. Валидация и обработка ошибок
- внедрила валидацию через Zod
- настроила отображение ошибок на frontend
- улучшила UX при вводе данных

### 6. Widget Engine и игра (Async Sorter)
Я разработала собственный обучающий виджет:
- продумала архитектуру
- реализовала backend API для уровней
- создала 30 уровней с усложнением
- разработала frontend-часть игры

Реализовала:
- drag-and-drop механику
- мини-игру выбора порядка выполнения
- оптимизацию рендера через useRef
- плавные анимации (transform, ghost element)

Также улучшила UX:
- подсветка элементов
- визуализация кода
- экраны победы/поражения
- логика усложнения уровней

Также я разработала расширяемый `WidgetEngine`, который позволяет легко добавлять новые виджеты и игры в проект:

- продумала модульную архитектуру для поддержки разных типов виджетов
- реализовала универсальные backend-эндпоинты для всех будущих виджетов
- обеспечила взаимодействие frontend и backend через единый API
- сделала структуру данных и уровней гибкой для расширения и масштабирования
- подготовила возможность интеграции новых визуальных компонентов и игровых механик без переписывания существующего кода

Это позволило не только создать Async Sorter: Event Loop Game, но и заложить основу для добавления других обучающих компонентов в будущем.

### 7. Stats
Я разработала полноценный сервис статистики для приложения, который собирает, хранит и отображает прогресс пользователей как по отдельным виджетам, так и в глобальном масштабе.

**Backend-сервисы:**

- Создала `StatsService` на NestJS с эндпоинтами для работы со статистикой
- Реализовала **Widget Stats API**:
  - получение статистики по конкретному виджету (последний пройденный уровень, лучшее время, общий прогресс)
  - обновление последнего уровня в зависимости от режима (старт или следующий уровень)
- Реализовала **Global Stats API**:
  - получение глобальной статистики пользователя (общая информация по всем виджетам)
  - обновление ежедневного стрика и даты последнего визита
  - обновление лучшего глобального времени прохождения
  - получение глобальной таблицы лидеров всех пользователей

## Личные Feature Components

## 1. Custom Auth (Backend)

**Что реализовано:**
- Полностью кастомная система аутентификации на NestJS с JWT + bcrypt
- Access + Refresh токены
- Refresh token rotation и безопасное хранение в httpOnly cookie
- Logout с инвалидацией токена и очисткой cookie
- Защита маршрутов + Throttler (5 запросов/мин)
- Обработка ошибок и валидация через DTO
- Авторизация через Google OAuth с popup и отдельным HTML файлом (без передачи скрипта)
- Связь Google аккаунта с существующим пользователем
- Сообщение при попытке логина через email для Google-пользователя

**Личный вклад / Особенности:**
- Продумала и реализовала архитектуру бэкенда, включая логику токенов и защищённые маршруты
- Сделала отдельный HTML-файл для Google OAuth для повышения безопасности
- Настроила Throttler для защиты эндпоинтов
- Проработала edge cases и ошибки:
  - Race condition при refresh токенах - внедрила grace period (2 минуты) и rotation токенов
  - Пользователь без пароля (Google OAuth) - отдельный режим «Set Password»
  - Повторная регистрация email - проверка перед созданием
  - Некорректные данные (email, пароль, пустые поля) - валидация через DTO + UI уведомления
  - Безопасность popup OAuth - отдельный HTML файл вместо отправки скрипта
- **Разбирала разные подходы хранения информации о пользователе:**
  - payload JWT (access токен) - быстрый доступ, но статический payload
  - хранение userId в Zustand на frontend - гибкая синхронизация, возможность сброса при logout/delete

## 2. Profile Page (Backend + Frontend)

**Что реализовано:**
- Полноценная страница профиля на React, интегрированная с бэкендом
- Кастомные хуки для работы с API: получение профиля, обновление данных, смена пароля, удаление аккаунта, загрузка аватара
- Валидация данных через Zod и управление состоянием через Zustand
- Управление черновиком (draft) + отмена изменений
- Интеграция с Supabase для хранения аватаров с публичной ссылкой
- Бэкенд-сервис на NestJS: отдельный UsersService для работы с пользователями, включая обновление профиля, смену пароля и аватара

**Личный вклад / Особенности:**
- Создала кастомные хуки для всех операций с профилем и валидацией данных
- Настроила хранение userId и ссылки на аватар в Zustand для глобального состояния
- Реализовала UX:
  - блокировка кнопки Save при отсутствии изменений
  - показ ошибок в UI
  - модальное окно «Set Password» для Google-пользователей
- Проработала edge cases:
  - некорректные данные
  - отсутствие данных
  - неверный пароль
- Интегрировала фронтенд с бэкенд-сервисом
- Адаптивная верстка, Tailwind, CSS variables для светлой/тёмной темы
