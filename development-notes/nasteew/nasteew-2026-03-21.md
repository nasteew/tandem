## Что было сделано
Вчера была сделана система сохранения статистики как для конкретной игры, так и для глобальной статистики пользователя. В рамках игровой статистики теперь фиксируются пройденные уровни, обновляется общее количество завершённых уровней и сохраняется время прохождения, а также сохраняется текущий уровень. Если игрок проходит уровень быстрее, чем раньше, система корректно обновляет лучшее время. Возникла проблема с обновлением последнего уровня, который проходит пользователь, но она была решена с помощью введения разного мода для кнопки start и для кнопки next, от того зависит сохранение текущего или следующего уровня.

Также была внедрена глобальная статистика пользователя, которая работает на уровне всего приложения. На фронтенде при входе в приложение была добавлена логика обновления streak — количество дней подряд, когда пользователь заходил в игру.

Были созданы и подключены необходимые API‑методы, обеспечивающие работу обеих систем. Для глобальной статистики реализованы методы получения данных, обновления streak, обновления лучшего времени и получения глобального списка пользователей, отсортированного по глобальным данным. Для игровой статистики добавлены методы сохранения факта прохождения уровня, обновления общего количества завершённых уровней через глобальный сервис статистики и обновления лучшего времени прохождения уровня также через глобальную статистику. Также появилась возможность получать глобальную  и игровую статистику по запросу для использования этих данных на дашборде.
