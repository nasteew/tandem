# Дата: 2026-03-29

## Что было сделано

**1. Страница глобальной статистики**

- Реализована страница `/statistic` с отображением данных всех игроков.
- Добавлены компоненты: `GlobalMetrics` (общая статистика), `LeaderboardHighlights` (топ-3 по стрику и уровням), `StatsTable` (таблица с сортировкой по столбцам).
- Настроена серверная сортировка через query-параметры (streak, levels, time).
- API‑слой: `statistic.api.ts`, хук `useGlobalStats` на React Query.
- Типы: `GlobalStatsUser`, `SortBy`.
- Адаптивность: таблица с горизонтальным скроллом, карточки метрик адаптируются.

**2. Обновление Dashboard после прохождения уровня**

- В мутациях `useLevelStats` и `useUpdateLastLevel` добавлен `queryClient.invalidateQueries({ queryKey: ['dashboard'] })` после успешного сохранения прогресса.
- Теперь данные на Dashboard обновляются без перезагрузки страницы.

**3. Исправление лишних запросов refreshToken для новых пользователей**

- В `AuthInitializer` добавлена проверка флага `wasLoggedIn`. Если его нет, `refreshToken()` не вызывается – устранены 401 ошибки для новых пользователей.

**4. Новые виджеты: Quiz и True/False**

- **Quiz** – выбор одного ответа из нескольких. Компонент `QuizWidget.tsx` с таймером (60 сек) и роботами (HappyRobot / AngryRobot). Уровни и решения созданы для easy, medium, hard (по 5 каждого).
- **True/False** – утверждение, которое нужно оценить как правда или ложь. Компонент `TrueFalseWidget.tsx` с двумя кнопками, мгновенной обратной связью и пояснением.
- Оба виджета интегрированы в `WidgetEngine` через новые `case 'quiz'` и `case 'true-false'`.
- Бэкенд: добавлены типы в `levels.service.ts` и `types/index.ts`, обновлён `widgets-meta.json`.

**5. Исправление мигания кнопки «SIGN IN» → «LOG OUT»**

- В `Navbar.tsx` добавлена проверка `isInitialized` из стора. Пока инициализация не завершена, блок авторизации остаётся пустым – кнопка не моргает.

**6. Исправление отображения карточек на Dashboard после регистрации**

- В `useUserGames` исправлена логика ожидания загрузки виджетов – `selectedGameSafe` больше не был `null` после первого входа, и все карточки отображаются сразу.

## Проблемы и решения

- **Лишние запросы на обновление streak на Dashboard** – использовала `useRef`, чтобы мутация вызывалась только один раз после монтирования (решение из предыдущего опыта).
- **NaN в прогрессе при отсутствии totalLevels** – добавила защиту: если `totalLevels` не пришёл или равен 0, использую значение по умолчанию 1.
- **Мигание кнопки авторизации** – решила через проверку `isInitialized`, чтобы рендер блока происходил только после того, как мы точно знаем статус пользователя.
- **Несколько итераций по массиву в `GlobalMetrics`** – в код-ревью предложили объединить вычисления в одном `reduce` для оптимизации, но для текущего размера данных оставила как есть (читаемость важнее).
- **Повторяющийся код инвалидации Dashboard** – вынесла в отдельную функцию `invalidateDashboard`, чтобы не дублировать.
