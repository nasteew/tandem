- **Что было сделано:**
  Реализовала страницу Dashboard с отображением статистики пользователя и прогресса по виджетам. Интегрировала бекенд‑API, добавила API‑слой (dashboard.api.ts) и хуки на React Query. В процессе столкнулась с несколькими проблемами, которые удалось решить.

- **Проблемы и решения:**

1. Множественные вызовы updateStreak
   На странице Dashboard начали бесконечно сыпаться запросы на обновление streak (из‑за эффекта в useEffect, который срабатывал при каждом рендере). Исправила, добавив useRef, чтобы мутация вызывалась только один раз после монтирования компонента.

2. Обернула методы login, register в try/catch в auth.ts – это предотвратило падение приложения при сетевых ошибках.

3. Лучшее время отображается как 0:00
   Для полного решения нужно будет доработать AsyncSorterGame: передавать (initialTime - remainingSeconds) \* 1000 вместо remainingSeconds.

4. NaN в прогрессе
   При отсутствии totalLevels от бекенда прогресс становился NaN. Добавила защиту: если totalLevels не пришёл или равен 0, использую значение по умолчанию (1).

5. Стили Dashboard
   Убрала избыточные градиенты и анимации, переделала на более сдержанный дизайн, переиспользовала существующие компоненты Card и Button.
